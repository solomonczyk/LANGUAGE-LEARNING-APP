# MVP Scope and Cut Line

**Status:** CANONICAL  
**Version:** 1.0.0  
**Effective date:** 2026-06-10  
**Owner:** Product Owner  
**Change control:** Changes require a documented change request or ADR where specified.

## MVP objective

Prove that a learner can complete a personalized end-to-end learning loop and that a meaningful subset returns and pays for the experience.

## MVP must-have

- onboarding;
- learner profile;
- multidimensional diagnostic;
- Learning Entry Contract;
- personal narrative lesson;
- at least one visual or audio lesson mode before closed beta;
- text submission;
- controlled AI analysis;
- schema, linguistic, pedagogical, and policy validation;
- deterministic lesson completion;
- mastery evidence;
- review scheduling;
- minimal reward acknowledgement;
- analytics and audit;
- security and cross-user isolation;
- operator diagnostics;
- privacy and deletion boundary;
- mobile accessibility baseline.

## MVP cut line

The public MVP may contain no more than:

- one target language: English;
- two support languages initially: Ukrainian and Russian, subject to final content capacity;
- four core lesson modes;
- one learner role;
- one read-only operator role;
- one primary commercial offer;
- one payment geography strategy;
- one controlled AI provider configuration.

## Explicitly out of scope before public MVP decision

- teacher marketplace;
- live tutor scheduling;
- classroom administration;
- multiplayer;
- public leaderboards;
- social feed;
- avatar economy;
- video generation;
- AR/VR;
- open-ended agent tools;
- unrestricted AI chat;
- multiple subscription tiers;
- B2B dashboards;
- exam certification;
- native desktop apps;
- offline model inference.

## Scope freeze rule

Any feature not required to pass a stage gate is excluded. “Useful”, “interesting”, or “competitors have it” is not sufficient justification.

## Feature admission test

A feature enters the current stage only if:

1. it supports the stage hypothesis;
2. it has an owner;
3. it has acceptance criteria;
4. it fits the stage deadline and budget;
5. it does not weaken a safety gate;
6. removing it would materially reduce the value of the experiment.
