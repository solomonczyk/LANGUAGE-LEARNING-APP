## Index governance

Every document added to this tree MUST:
1. Have a unique numeric prefix (non-overlapping with existing)
2. Be listed in this index on creation
3. Have Status, Version, Purpose, In scope, Out of scope sections
4. Pass the canonical documentation validation script
5. Be registered in `artifact_index.json`
6. Be linked in the Requirements Traceability Matrix if it contains requirements
